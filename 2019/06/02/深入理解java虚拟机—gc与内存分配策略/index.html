深入理解java虚拟机 — gc与内存分配策略

《深入理解java虚拟机》读书笔记
java基础
发布日期:   2019-06-02

内存运行时分区：程序计数器，虚拟机栈和本地方法栈，都会随线程的销毁而回收

因为主要需要GC的是堆和方法区

名词解释：在 Java 中，堆被划分成两个不同的区域：新生代 ( Young )、老年代 ( Old )。新生代 ( Young ) 又被划分为三个区域：Eden、From Survivor、To Survivor。

80%eden 10%survivor

对象存活判定算法

1 引用计数算法：有引用就加1，引用失效减1，判断对象是否存活

2 可达性分析算法： 以 GC ROOTS 的对象为起始点，向下搜索节点，搜索所走过的路径称为引用链。如果一个对象 到 GC ROOTS 没有引用链相连，那么认为此对象不可用。(主流商业用这种)

可以作为GC ROOTS对象的：

  1. 虚拟机栈中引用的对象
  2. 方法区中类静态属性引用的对象
  3. 方法区中常量引用的对象
  4. 本地方法栈中JNI(native方法)引用的对象

引用类型：

强引用 软引用 弱引用 虚引用

finalize方法

方法区是老年代

垃圾收集算法

1 标记-清除算法(老年代)

先标记所有需要回收的对象，然后统一完成清除回收。但标记和清除的效率都不高，且会造成很多不连续的内存碎片。当需要分配较大的对象时，会因为无法找到连续的空间而再一次触发垃圾收集动作

2 复制算法(新生代)

将内存按容量分为大小相同的两块。然后将已使用的对象复制到另一块，再将这块空间一次性清理，这样效率比较高，每次直接对一半的内存区域清理，没有碎片。代价就是将内存缩小为一半。(商业虚拟机都用这种算法来回收新生代)

3 标记整理法(老年代)

复制算法在对象存活率较高时，就会进行比较多的复制操作，效率就会降低。

和标记清除的标记部分一样，但不直接对可回收对象进行清理，而是让存活的对象向一端移动，然后直接清理掉端边界以外的内存。

4 分代收集算法

分老年代和新生代。新生代用复制算法，老年代用标记清除或者标记整理

具体算法实现

1 枚举根节点

2 安全点

安全点的特征是”长时间执行”，指令序列复用，如方法调用，循环跳转，异常跳转等

如何让线程都跑到安全点。分抢先式中断和主动式中断。

抢先式：GC让所有线程中断，没到安全点的，让跑到安全点再中断

主动式：GC设置一个标志，所有线程去轮询这个标志，中断标志为true就中断

1 serial收集器 新生代

复制算法，并行的多线程收集器

可控制吞吐量，停顿时间越短，交互越好

吞吐量 = 运行用户代码时间 / (运行用户代码时间 + 垃圾收集时间)

内存分配与回收策略

  1. 对象主要分配在堆的新生代 的eden

新生代GC 是 minor gc

老年代GC 是major、full gc。

当出现major gc ，至少伴随一次minor gc。且major gc比minor gc慢十倍以上

  1. 大对象直接进入老年代

    大对象是指需要大量连续内存空间的java对象，例如byte[]这种，很长的字符串。

  2. 长期存活的对象进入老年代

给对象进行标记，minor gc一次，还存活的，就年龄+1，达到设置的值就进入老年代了

  1. 动态对象年龄判定

在survivor空间中相同年龄所有对象大小的总和大于survivor空间的一半，年龄大于或等于该年龄的对象可以直接进入老年代，不需要等 Max-TenuringThreshold中要求的年龄

  1. 空间分配担保

在minor gc 之前， 新生代的survivor无法容纳的对象需要进入老年代，老年代需要担保 容纳 晋升到老年代中对象的空间。通常取平均值作为经验值

结论：只要老年代的连续空间大于新生代对象总大小或者历次晋升的平均大小，就会进行minor gc，否则进行full gc
